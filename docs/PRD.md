# 产品需求文档（PRD）— 同事常用工具箱

## 1. 项目定位

基于 **Streamlit** 的轻量 Web 工具集合，面向公司内部同事日常使用。部署在 Streamlit Cloud 或内网服务器，通过浏览器访问，无需安装客户端。

## 2. 用户与场景

- **用户**：需要快速完成格式转换、富文本清理等工作的内容、运营、开发同事。
- **场景**：临时处理文本/HTML，复制结果到邮件、CMS、文档或代码中。

## 3. 非目标（当前阶段不做）

- 用户账号、权限与审计。
- 服务端持久化存储（不上传、不保存用户输入到数据库）。
- 完整 HTML 安全消毒（XSS 防护、CSP 等）；工具仅做结构/标签层面的约定转换。

## 4. 信息架构与导航

- **首页**：类应用启动器的方形入口按钮，点击进入对应工具。
- **工具页**：顶部提供「返回首页」，通过 `st.session_state` 切换视图。
- **入口**：仓库根目录 `app.py` 为 Streamlit 主入口（Cloud 默认 `streamlit run app.py`）。

## 5. 工具清单

### 5.1 数学斜体转换器

- **目标**：将输入中的英文字母转为 **Mathematical Italic** 对应的 Unicode 实体（`&#x…;`），便于在只支持 HTML 实体的环境中显示。
- **输入**：单行英文文本（其他字符原样保留）。
- **输出**：实体字符串，可复制。

### 5.2 HTML / div 修复（清理）

- **目标**：对粘贴的 HTML 做轻量规范化，便于在只接受段落/强调标签的编辑器中使用。
- **规则**：
  - `div` → `p`（标签名替换，保留属性）。
  - `h4` → `p` 内包一层 `strong`（保留原 `h4` 内所有节点）。
  - `i` → `em`。
  - 删除仅含空白或 `&nbsp;`、且无子标签的 `p`。
- **实现**：正则替换标签名 + `BeautifulSoup`（`html.parser`）处理 `h4` 与空 `p`；**不**对 `ul`/`li` 做扁平化或重排，以降低列表结构被破坏的风险。
- **限制**：异常嵌套、复杂脚本/样式节点可能需要人工复查；仅含 `<br>`/图片等子元素的「空」段落删除规则见后续迭代说明。

## 6. 技术约束

- **语言**：Python 3。
- **依赖**：见根目录 `requirements.txt`（`streamlit`、`beautifulsoup4`）。
- **结构**：`app.py` 为薄入口；页面在 `ui/`；纯逻辑在 `utils/`；说明文档在 `docs/`。
